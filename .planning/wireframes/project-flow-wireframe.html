Wireframe visual / decision de flujo

Home lineal vs. home + project list.

Dos recorridos complementarios para el cierre del portafolio: uno rapido para contratar por criterio, otro exploratorio para revisar profundidad sin depender del footer.

lectura rapida exploracion mobile first calidad premium
Opcion A / Estado actual refinado

Una sola pagina editorial.

El index hace casi todo: presenta criterio, selecciona proyectos clave, entrega confianza y conduce a contacto. Es la ruta mas eficiente para visitantes con poco tiempo.

A
01
Index / above the fold

Hero con impacto inmediato.

Nombre, disciplina, disponibilidad, frase editorial y entrada visual controlada.

02
Criterio

Indice, manifiesto y clientes.

Construye confianza antes de mostrar volumen. Ideal para una lectura lineal.

03
Trabajos seleccionados

VI Summit, ConSentido, AMXiTech.

Pocas piezas, alta curaduria. Cada bloque debe sentirse premium en mobile y desktop.

04
Prueba

Approach, quote, experiencia.

La contratacion se apoya en proceso, autoria honesta y consistencia visual.

05
Cierre

Contacto + footer.

Funciona si el visitante siguio el relato. Riesgo: quien quiera explorar mas depende del pie.

Footer como ultimo mapa
Opcion B / Index + pagina aparte

Home editorial + directorio.

El index mantiene la narrativa rapida, pero suma un CTA sobrio despues de trabajos. La pagina aparte sirve como mapa interactivo para clientes o estudios que quieren revisar profundidad.

B
01
Index / mismo impacto

Hero, criterio, casos clave.

No se reduce la magia del home. El directorio aparece como expansion, no como reemplazo.

02
Micro CTA

Explorar archivo de proyectos.

Una franja pequena bajo trabajos seleccionados. Invita a profundizar sin cortar el scroll lineal.

CTA recomendado: "ver mapa de proyectos"
03
/projects

Directorio como sistema editorial.

Filtros por disciplina, casos destacados y lista densa. Debe sentirse curado, no catalogo generico.

04
Decision assist

Ruta por intencion.

Branding completo, web, contenido, eventos, proyectos bajo direccion o colaboracion.

Cada item puede abrir caso, preview o contacto
05
Cierre doble

Caso puntual o contacto.

El usuario puede volver al home, abrir un caso o contactar desde una decision mas informada.

Footer deja de ser la unica navegacion profunda
Lectura de estrategia

La mejor version no es elegir entre narrativa y exploracion. Es mantener el index como recorrido de contratacion rapida y dejar `/projects` para la persona que necesita pruebas, comparacion y profundidad. Por eso el directorio queda al final del roadmap: primero se pule mobile, motion e imagen; despues se decide si el mapa suma sin diluir.
